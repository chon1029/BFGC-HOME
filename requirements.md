## 📌 프로젝트 개요

### 배경
- **현재 상황**: WordPress + Elementor Pro로 제작된 홈페이지 (www.bfgc3.mycafe24.com)
- **문제점**:
  - Elementor 플러그인의 모바일 반응형 이슈
  - 플러그인 간 충돌로 인한 화면 깨짐
  - 느린 로딩 속도
  - 유지보수 어려움
- **해결 방안**: Next.js 14 + Sanity (Headless CMS) + Vercel 배포로 완전 재구축

### 프로젝트 목표
1. **반응형 완벽 지원**: 모든 디바이스에서 깨지지 않는 UI
2. **강력한 임팩트**: 방문자에게 강렬한 첫인상 제공
3. **모던한 디자인**: 최신 웹 트렌드 반영
4. **빠른 성능**: 3초 이내 로딩
5. **쉬운 유지보수**: 플러그인 의존도 제거

## 🎨 디자인 요구사항

- **컬러**:
  - Primary: `#0c076d` (보라/남색)

### UI/UX 원칙
1. **모바일 퍼스트**: 모바일에서 완벽, 데스크톱으로 확장
2. **직관적 네비게이션**: 3클릭 이내 모든 정보 접근
3. **명확한 CTA**: 중요 행동 유도 버튼 강조
4. **일관성**: 모든 페이지 동일한 디자인 시스템
5. **접근성**: WCAG 2.1 AA 준수

### 애니메이션 요구사항
- **페이지 로드**: 요소들 순차적 페이드인
- **스크롤**: 스크롤 트리거 애니메이션
- **호버**: 버튼, 카드 호버 효과
- **전환**: 페이지 간 부드러운 전환
- **성능**: 60fps 유지, 모바일 배터리 고려

## 🔧 기술적 요구사항

### 성능
- **로딩 속도**: 초기 로드 <3초 (3G)
- **Lighthouse 점수**:
  - Performance: 90+
  - Accessibility: 90+
  - Best Practices: 90+
  - SEO: 90+
- **이미지 최적화**: WebP, lazy loading
- **코드 분할**: Route-based code splitting

### 반응형
- **브레이크포인트**:
  - Mobile: 0-767px
  - Tablet: 768-1023px
  - Desktop: 1024px+
- **테스트 디바이스**:
  - iPhone (Safari)
  - Android (Chrome)
  - iPad (Safari)
  - Desktop (Chrome, Firefox, Safari, Edge)

### SEO
- **메타 태그**: 모든 페이지 고유 title, description
- **Open Graph**: SNS 공유 최적화
- **Sitemap**: XML sitemap 생성
- **Robots.txt**: 검색 엔진 최적화
- **구조화된 데이터**: Schema.org 마크업 (교회, 이벤트 등)

### 접근성
- **Semantic HTML**: 의미있는 HTML 태그
- **ARIA**: 스크린 리더 지원
- **키보드 네비게이션**: Tab으로 모든 요소 접근
- **색상 대비**: WCAG AA 기준 충족
- **Alt 텍스트**: 모든 이미지 설명

### 보안
- **HTTPS**: SSL 인증서 필수
- **폼 유효성**: 클라이언트/서버 검증
- **XSS 방지**: 사용자 입력 sanitize
- **CSRF 방지**: 토큰 기반 보호

## 📊 콘텐츠 관리

### 정적 콘텐츠
- **페이지 텍스트**: 코드에 직접 작성 (Markdown 또는 JSON)

- **주보**: PDF 파일 Sanity Asset으로 관리
- **갤러리**: Sanity Image Pipeline 사용 (자동 최적화)
- **일용할양식**: Sanity Editor로 작성 및 관리
- **기도제목**: Sanity Document로 관리

### 폼 데이터 처리
- **새가족 등록**: 이메일 전송 (Resend, SendGrid)

- **문의**: 이메일 전송
- **헌금**: 외부 결제 시스템 연동
